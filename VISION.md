# DrugMind v2.0 — 药物研发人员的数字分身协作平台

> **GitHub + Second Me × 药物研发 = 所有制药人都能在上面协作的平台**

## 🎯 产品定位

**一句话**：让每一位药物研发人员拥有自己的AI数字分身，7×24与其他研发人员的分身讨论药物项目。

**类比**：
- Second Me = 通用数字分身（所有人都能用）
- ChatDev = 通用AI团队协作（写代码的）
- **DrugMind = 药物研发垂直的数字分身协作平台**

| 用户群 | 痛点 | DrugMind价值 |
|--------|------|-------------|
| 药企研发人员 | 跨部门沟通慢，会议多 | 数字分身7×24在线讨论 |

## 🏗️ 平台架构

### 用户系统
```
用户注册 → 选择专业领域 → 训练数字分身 → 开始协作
```

- **个人档案**：姓名/机构/专长/发表论文/项目经验
- **数字分身**：基于Second Me训练，保留个人判断风格
- **分身广场**：公开/私有，可被其他人"问问题"

### 角色体系（扩展）
不再只有5个预设，支持用户自定义：

**基础角色**（预设模板）：
- 药物化学家 / 生物学家 / 药理学家 / 数据科学家 / 临床医生
- 项目负责人 / 专利律师 / 法规专家 / 商业BD

**自定义角色**：
- 用户可创建任意角色（如"GLP-1制剂专家"）
- 训练数据：上传论文/实验记录/项目文档
- 人格设定：保守/激进/平衡的判断风格

### 协作模式

#### 1. 单问（Ask）
向某个专家的分身提问 → 获得该专家视角的回答
```
你 → 张博士的分身 → "这个SA Score=7.2你怎么看？"
```

#### 2. 团队讨论（Team）
创建虚拟团队，围绕议题讨论
```
你 → 创建讨论"CPD-47要不要进先导优化"
   → 邀请：张博士(药物化学) + Dr.Smith(药理) + 李教授(数据)
   → 分身们轮流发言 → 形成共识
```

#### 3. 项目空间（Project）
为具体药物项目创建专属空间
```
项目："MG新药AChE抑制剂"
成员：5个数字分身 + 3个真人
看板：靶点确认 → 虚拟筛选 → Hit-to-Lead → 先导优化 → 候选
记录：所有讨论/决策/数据自动归档
```

#### 4. 分身网络（Network）
- 关注你感兴趣的专家分身
- 分身会"学习"被问过的问题
- 热门分身有"声誉值"
